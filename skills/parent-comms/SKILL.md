# KP13 Parent Communications

## When to Use

- Weekly progress report to a player's parent
- Any message that a parent (not the player, not the coach) will read
- Between-session updates that need to make a parent feel "this person knows my kid and has a plan"

## The Lagoon Oasis Principle

**The job is to make the parent feel confident and secure** — not to impress them with detail, not to defend the methodology, not to upsell. The parent who finishes reading should think: *"Kasper sees my child clearly, has a plan, and is moving them forward."*

What this looks like in practice:
- **One clear plan, not 50 options.** State what we worked on and what's next — don't list every possibility we considered.
- **Show progress over time.** Anchor each report to where the player was, not just where they are.
- **Be transparent about methodology.** When you reference EPM scores or development stages, briefly explain what they mean. Never hide behind jargon.
- **Honest, not defensive.** If the week was rough, say so plainly and say what we're doing about it.

## Joy-First Development Language

KP13 develops the **whole footballer** — technical, tactical, mental. Joy is part of the model, not a soft add-on. If joy or engagement dipped this week, **flag it explicitly**. A parent should never read a glowing report and then hear from their kid that training felt miserable.

Use **stage language** over raw numbers where possible:
- *"Marius is moving from Developing to Confident in first touch"*  ✅
- *"Marius scored 6.2 in first touch"*  ❌ (only use numbers when concrete tracking matters)

## Danish Register

- Write in **Danish**, warm but direct.
- Address the parent as the audience; refer to the player by first name throughout.
- Avoid English coaching jargon. If you must use a technical term (rondo, first touch, push-and-go), use the Danish-context phrasing the player would hear in training.
- No emojis. No exclamation marks for emphasis. The tone is calm and professional, like a trusted private tutor — not a hype account.

## Required Structure

Every weekly parent message must contain these four sections, in order, with these exact Danish headings:

```
## Ugen der gik for [navn]

### Hvad vi arbejdede på
Kort opsummering af ugens sessioner — én sammenhængende paragraf.

### Hvad jeg så
Konkrete observationer, specifikke øjeblikke, fremskridt. Brug stage-sprog.
Hvis glæde eller engagement var lavt, så sig det her, direkte.

### Hvad vi gør næste uge
Hvor vi er på vej hen — én eller to sætninger. Knytter an til hvad jeg så.

### Sådan kan du hjælpe derhjemme
Én konkret, nem ting forælderen kan gøre — ikke en liste, ikke en træningsplan. Én ting.
```

## Length & Tone Rules

- **250-350 words total.** Longer than this and the parent skims. Shorter and it feels lazy.
- **Be honest but encouraging.** Never sugarcoat a regression — but always frame it as part of the plan.
- **Specific moments beat generic praise.** "Marius modtog bolden på halv-vending tre gange i træk på fredag" beats "Marius er blevet bedre til first touch."
- **Never reference internal EPM dimension keys** (e.g., `first_touch`, `decision_speed`). Translate to plain Danish.
